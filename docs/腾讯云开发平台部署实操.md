# 腾讯云开发平台部署实操（当前项目）

本文对应项目路径：
- 后端：`/Users/apple/Downloads/专升本/backend`
- 小程序：`/Users/apple/Downloads/专升本/wechat-miniprogram`
- 老师后台静态页：`/Users/apple/Downloads/专升本/teacher_admin.html`

## 1. 目标架构

1. 云托管（CloudBase Run）部署 NestJS 后端。
2. PostgreSQL 存题库、学生、做题记录。
3. 小程序调用云托管 API。
4. 老师后台页面走静态托管，调用同一套后端 API。

## 2. 第一步（现在就做）

在腾讯云开发控制台创建云托管服务：

1. 进入「云开发 CloudBase」-> 选择环境 -> 「云托管」-> 新建服务。
2. 代码来源选你的仓库（或上传代码）。
3. 服务目录选择 `backend`。
4. 构建方式选 `Dockerfile`（项目已提供：`backend/Dockerfile`）。
5. 监听端口填 `3000`。
6. 实例最小规格可先用 0.5C1G（内测阶段）。

## 3. 环境变量（云托管里配置）

至少配置以下变量：

1. `DATABASE_URL=postgresql://...`
2. `JWT_SECRET=请用强随机串`
3. `JWT_EXPIRES_IN=7d`
4. `ADMIN_SECRET=老师后台调用密钥`
5. `PORT=3000`
6. `CORS_ORIGIN=https://你的老师后台域名`

说明：
- 如果小程序也直连该 API，`CORS_ORIGIN` 对小程序不影响，但对网页老师后台有影响。
- `ADMIN_SECRET` 只给老师后台页面使用，不要写进小程序代码。

## 4. 初始化数据库（一次性）

建议在本地执行（连接到腾讯云 PostgreSQL）：

```bash
cd /Users/apple/Downloads/专升本/backend
npm install
npm run prisma:generate
npm run prisma:migrate -- --name init
npm run seed
npm run import:students -- --file=./scripts/students_template.csv
```

## 5. 小程序接入

1. 修改小程序后端地址：  
`/Users/apple/Downloads/专升本/wechat-miniprogram/config.js`

```js
module.exports = {
  API_BASE_URL: 'https://你的云托管域名/api/v1',
  REQUIRE_AUTH: true
};
```

2. 在微信公众平台配置「request 合法域名」为你的后端 HTTPS 域名（不带路径）。

## 6. 老师后台接入

1. 将 `teacher_admin.html` + `teacher_admin.js` 放到静态托管（CloudBase 静态网站托管或 COS）。
2. 打开页面后填写：
- `API Base`：`https://你的云托管域名/api/v1`
- `Admin Secret`：你在后端环境变量配置的 `ADMIN_SECRET`

## 7. 上线前检查

1. `POST /api/v1/auth/register` 能注册白名单学生。
2. `GET /api/v1/questions` 未登录返回 401。
3. 学生冻结后，旧 token 访问受保护接口返回 403。
4. 老师后台能正常加载学生列表、冻结、改设备上限。

## 8. 中国大陆上线注意项

1. 使用 HTTPS 域名（小程序强制）。
2. 若用自定义域名对公网提供服务，按监管要求完成备案流程。
3. 隐私政策、用户协议、账号注销路径要在小程序提审前准备好。
